02 Architecture Decisions & Trade-offs
D-1 JWT (stateless) vs Session (stateful) Chose: JWT
Rationale Stateless 架构, 零成本水平扩展, 无需引入 Redis 依赖
Trade-off Token 签发后无法立即撤销；如需强制登出功能，后续必须实现 token blacklist
Risk 若业务未来要求实时吊销能力，需引入 Redis/DB 查询，部分抵消 stateless 优势
Do you agree with this decision?
D-2 bcrypt (cost=12) vs argon2id Chose: bcrypt
Rationale 成熟生态、无 native compilation 依赖，跨平台兼容性更优
Trade-off 在同等安全级别下比 argon2id 慢约 40%
Note 已评估 argon2id，但 node-argon2 在 Alpine Linux 上存在构建问题，影响 Docker 部署
Do you agree with this decision?
D-3 Email link vs Verification code (密码重置) Chose: Email link
Rationale 更顺畅的用户体验，无需手动输入验证码
Trade-off 依赖邮件送达率；邮件客户端可能预取链接导致 token 提前消耗
Mitigation 1h 过期 + 一次性使用 + 防预取 POST 确认页（链接打开后需点击确认按钮）
Do you agree with this decision?
